질문 : Pygame에서 이미지와 충돌 박스(AABB, Circle)를 어떻게 매칭하고 화면에 그리나요?
대답 : pygame.Rect를 이용한 사각형 범위와 중심점 기준의 반지름(radius)을 설정하여 시각화함.
질문 : 오브젝트를 회전시키고, 특정 키(Z)를 누를 때 회전 속도를 높이려면?
대답 : Vector2.rotate()를 활용해 모서리 좌표를 실시간 계산하고, 조건문을 통해 가변적인 rotation_speed를 적용함.
질문 : 회전하는 사각형에 딱 맞는 충돌 판정(OBB)을 어떻게 구현하나요?
대답 : 분리축 정리(SAT)를 도입하여 두 다각형 사이의 투영 영역이 겹치는지 판단하는 정교한 물리 로직 완성.

# 원형 vs AABB
수직/수평 접근: 오브젝트의 면으로 접근할 때는 원형과 AABB가 거의 동시에 충돌한다.

대각선(모서리) 접근: AABB는 사각형의 모서리 끝에서 먼저 충돌이 발생하지만, 원형은 모서리 부분이 깎여 있어 더 깊숙이 들어가야 충돌이 발생함. 
즉, 원형 충돌은 모서리 부근에서 판정 공백이 생긴다.

# AABB vs OBB
회전 시의 변화: 오브젝트가 회전할 때 OBB는 크기가 일정하게 유지되지만, AABB는 회전하는 물체를 모두 포함하기 위해 크기가 계속해서 변한다.

정밀도 차이: 오브젝트가 45도 정도로 기울어졌을 때, AABB는 실제 물체가 없는 빈 공간까지 충돌 영역으로 잡는 반면
OBB는 물체의 회전각에 맞춰 정확한 외곽선에서만 충돌을 감지한다.

# AI와의 대화에서 배운 것
수학적 최적화: 원형 충돌 계산 시 루트 연산 대신 제곱 거리를 비교하는 것이 성능 면에서 유리하다는 점이 흥미로웠습니다.

# 내 게임에 적용한다면?
플레이어 캐릭터 및 보스 : 피격 판정이 예민한 주요 캐릭터에게는 OBB를 적용한다.
캐릭터가 회전하거나 무기를 휘두를 때 시각적 일치감을 높여 억울한 죽음이 없는 정교한 액션을 구현하고 싶습니다.
